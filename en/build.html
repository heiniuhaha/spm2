spm build focuses on package build, it is not designed as a build tool. It is based on grunt-spm-build which is a collection of grunt tasks.

Standard Package

The folder structure of a standard package:

package.json
src/
    filename.js

After spm build, the folder structure:

package.json
src/
    filename.js
dist/
    filename.js
    filename-debug.js

Learn how to write package.json.

Customize Build

spm build uses grunt as the task manager. If you want to customize and control the build process, you need to write your own grunt tasks.

spm build will call the build task in your Gruntfile.js.

An example of Gruntfile.js:

module.exports = function(grunt) {
    grunt.registerTask('build', function() {
        console.log('customize build')
    });
}

When spm build, it will print customize build.

Learn more about grunt-spm-build, and customize your own build process. You can even write your own build tools with the help of our low level api cmd-util.

Remote Gruntfile

Remote gruntfile is designed for team collaboration.

Add a gruntfile config in ~/.spm/spmrc:

[user]
gruntfile = http://your-company.com/gruntfile.js

When spm build, it will download the gruntfile, and run the build task in this gruntfile.
